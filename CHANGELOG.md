# Changelog

## 3.0.0 - 2026-05-26

### Added

- 新增 Heartbeat 控制，可保持默认、降频或在非编辑页禁用。
- 新增前台 feed/REST/XML-RPC 轻量控制。
- 新增 admin-ajax 诊断，统计登录/访客 AJAX Hook 与 Heartbeat 状态。
- 新增高级缓存就绪检查，检测 `WP_CACHE` 与 `advanced-cache.php`，但不自动写入 drop-in。
- 新增媒体库只读体检，展示附件总数、未挂载附件、缺少附件元数据。
- 新增插件/主题体检，提示缓存/优化类插件冲突风险。
- 新增性能趋势记录，保留最近 30 次诊断关键指标。
- 新增 WooCommerce/Action Scheduler 检测，展示待执行/失败任务和 Hook TOP。

### Safety

- 高级缓存只做就绪检测，不自动接管 `advanced-cache.php`。
- 媒体库体检只读，不自动删除未挂载附件。
- REST API 访客限制和 XML-RPC 禁用默认关闭，避免影响业务接口。

## 2.9.0 - 2026-05-26

### Added

- 新增 postmeta 深度治理模块。
- 支持检测并分批清理低风险 postmeta：`_edit_lock`、`_edit_last`、`_wp_old_slug`、`_oembed_*`。
- 支持检测并分批清理低风险字段中的完全重复 postmeta，每组保留最早一条。
- 新增超大 `meta_value` TOP 只读审查，避免盲删高风险数据。
- 新增 autoload 优化器，展示 autoload 总体积、大项、风险级别和保护状态。
- 支持将非保护 option 改为不自动加载，并保存回滚记录。

### Safety

- WordPress 核心关键 option、主题 mods、active_plugins、rewrite_rules、cron 等默认保护，不允许直接改为不自动加载。
- postmeta 深度清理仅处理低风险缓存/编辑痕迹字段，不自动删除业务字段或已发布文章。

## 2.8.0 - 2026-05-26

### Added

- 新增后台文章列表 AJAX 延迟统计。
- 支持 AJAX 延迟加载月份筛选，首屏不再同步执行月份聚合查询。
- 支持 AJAX 延迟加载精确总数，配合禁用 `found_rows` 后异步显示总数和页数。
- 延迟统计结果使用 transient 缓存，文章更新/删除/回收后自动清理。
- 设置页新增延迟统计开关与缓存时间配置。

## 2.7.1 - 2026-05-26

### Changed

- 页面缓存稳定性增强：记录输出缓冲层级，降低与其他主题/插件 buffer 冲突概率。
- 页面缓存目录自动创建 `index.html` 与 `.htaccess`，避免目录浏览并保留缓存目录保护文件。
- 后台页面缓存模块新增最后生成时间、最后清理时间。
- 页面缓存清理会记录元信息，保存设置后即时刷新诊断报告缓存。
- 文档补充：当前为轻量页面缓存模式，后续可升级 advanced-cache 高级模式。

## 2.7.0 - 2026-05-26

### Added

- 新增轻量页面缓存模块，默认关闭，避免与服务器级缓存重复冲突。
- 支持缓存首页、文章页/页面、分类/标签/归档页。
- 自动排除登录用户、后台、搜索、预览、REST/AJAX、带查询参数 URL 和常见登录/评论/Cookie 场景。
- 新增缓存 TTL、移动端/PC 分开缓存、缓存文件数/体积统计、响应头 `X-WLCO-Cache`。
- 支持发布/删除文章、评论变化自动清空页面缓存，并提供后台一键清空。
